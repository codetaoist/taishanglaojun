# 启动包清单与交付项（补齐）

目标：提供首批可交付的、可审可验的“启动包”结构与门禁，确保契约、样例、工具链与运维发布一体化。

## 目录结构
- `contracts/`：`openapi/laojun.yaml`, `openapi/taishang.yaml`
- `docs/`：开发手册、对齐规范、测试策略、运维与发布说明
- `plugins/`：示例插件清单与源码、签名与校验脚本
- `native/`：端侧模板工程（iOS/Android/Harmony/桌面）
- `tests/`：契约与端侧测试用例

## 交付项
- 契约与规范：接口契约、错误码、统一响应包装、版本与兼容策略
- 工程样例：后端控制器骨架、插件 SDK、端侧模板与示例组件
- 工具链：调试与观测、签名校验、依赖扫描、CI 集成脚本
- 发布与运维：灰度与回滚策略、权限与合规清单、监控与告警

## 验收门禁
- 契约测试通过：`schemathesis` 针对 `openapi/*.yaml` 报告无阻断
- 关键端侧场景可运行：插件 `install/start/stop/upgrade/uninstall` 主流程打通
- 清单签名与权限位校验通过：来源可信、权限最小化与资源配额明确
- 安全与合规审核通过：供应链扫描无阻断；隐私与数据收集说明完备
- 基础性能指标达标：接口吞吐与延迟、端侧 CPU/Mem/GPU/电量与 IO 指标在基线内

## 使用指引（简）
- 克隆后先阅读 `docs/` 中的流程与契约说明，按阶段执行 CI 验证
- 本地运行后端骨架与示例插件，端侧模板接入并完成 UI 集成验证
- 提交前生成签名与清单校验报告，附带测试覆盖与性能基线
