# 前端信息架构与契约对齐（React Admin）

以 OpenAPI 为契约，划分导航、页面与组件，保证与后端分域一致。

## 导航结构
- Laojun
  - Plugins（列表/安装/启停/升级/卸载）
  - Audits（审计日志查询）
  - Configs（系统配置）
- Taishang
  - Models（列表/新增/禁用）
  - Vectors（集合列表/新增/参数）
  - Tasks（提交/查询/取消/重试）

## 页面与数据契约
- 列表页：分页、过滤、排序；统一请求参数与响应结构。
- 详情页：资源详情加载；操作按钮遵循权限与审计策略。
- 表单页：字段校验与契约约束；错误码映射与提示。

## 状态管理与SDK
- OpenAPI 生成 TypeScript 类型与 SDK；统一异常处理与错误码。
- Store 切分：`laojun/*` 与 `taishang/*`；避免跨域耦合。

## 组件与UI
- 通用表格与表单组件；统一分页与过滤控件。
- 插件操作组件：安装/启停/升级的流程化UI，展示审计与结果。
- 任务监控组件：状态流转与重试/取消操作；资源限流提示。

## 权限与可见性
- 角色驱动导航与按钮可见性；前端拦截越权请求。
- 审计与提示：关键操作二次确认与审计条目链接。
