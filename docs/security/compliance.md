# 安全与合规策略

覆盖认证鉴权、插件签名校验、审计日志、机密管理与合规要求。[0]

## 认证与鉴权
- 认证：JWT/OIDC，所有接口需要 `Bearer` token。
- 鉴权：RBAC（角色-权限绑定），细分到资源与操作（安装/启停/升级/卸载）。
- 多租户（可选）：通过租户ID隔离数据与权限。

## 插件签名与清单校验
- 清单（manifest）：包含 `id`, `name`, `version`, `entry`, `permissions`, `checksum` 等。
- 签名：开发者私钥签名，服务端公钥校验；来源仓库白名单。
- 校验流程：清单结构 → 版本语义化 → 依赖检查 → 签名与校验和验证。

## 审计日志
- 记录维度：`actor`, `action`, `target`, `payload`, `result`, `timestamp`, `traceId`。
- 覆盖范围：插件安装/启停/升级/卸载、权限变更、配置变更。
- 合规保留：满足审计期存储与访问控制要求。

## 机密管理
- Env/Secrets：集中管理，敏感信息不入库不入日志。
- 轮转：密钥定期轮换；最小权限原则。

## 合规与隐私
- 隐私数据分类分级与访问控制；脱敏与加密传输。
- 合规参考：GDPR/CCPA（按业务需要），数据保留策略与删除策略。
