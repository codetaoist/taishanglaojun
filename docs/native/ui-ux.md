# 原生前端 UI/UX 设计规范（扩充版）

确保各终端保持原生体验一致性，同时遵守平台规范。[0]

## 通用原则
- 信息层级清晰；关键操作前确认；可回滚提示。
- 状态可见：加载/错误/成功/进行中均有反馈。
- 无阻塞：弱网/离线下保留可用操作与提示。

## iOS（SwiftUI）
- 导航：`NavigationStack` 清晰分层；Sheet/Modal 场景控制。
- 交互：手势与触觉反馈一致；Dark Mode 支持。
- 组件：列表分页与搜索；按钮语义化；图标遵循 SF Symbols。

## Android（Compose）
- 导航：`NavHost` 与 Back-stack 合理；状态管理与生命周期一致。
- 交互：Material Design 3；触觉与动画一致。
- 组件：列表与卡片；权限提示与后台服务状态展示。

## 机器人端（Qt/QML）
- 低延迟：减少复杂动画；大按钮与高对比度；紧急操作突出。
- 状态：实时刷新与断网提示；日志与审计入口清晰。

## 手表端
- 卡片与通知优先；短交互与快速完成；避免文本输入。

## 设计基线与设计令牌
- 色彩/字体/间距：统一设计令牌（tokens）并按平台微调；暗黑模式优先。
- 组件一致性：跨端确保同一组件语义一致，交互遵循平台规范。

## 导航与交互模式
- iOS：UINavigationController/UITabBar；手势返回与系统交互惯例。
- Android：Toolbar/BottomNavigation；系统返回键与深层导航栈管理。
- Harmony：分布式导航与多设备协同的栈切换策略。
- 桌面：菜单/快捷键/窗口管理；多窗口与拖拽操作一致性。

## 输入与反馈
- 键盘与输入法：输入框焦点/键盘弹出与遮挡处理；
- 触控与鼠标：点击、长按、右键、滚轮差异；提供可发现的反馈。
- 反馈等级：轻提示（Toast）、确认（Dialog）、错误（Banner/Inline）。

## 可访问性与国际化
- 可访问性：语义化控件、标签关联、对比度与可读性；适配 VoiceOver/TalkBack。
- 国际化：文本长度伸缩与双向文字；日期/货币/度量单位格式差异。

## 性能与动画
- 动画：轻量、可中断，避免复杂自定义导致掉帧；硬件加速优先。
- 列表与图片：虚拟化、占位与渐进加载；缩略图与缓存策略。
